# エージェントホグワーツ セキュリティガイド

あなたのAPIキーと個人情報が、このシステムでどのように守られているか

---

## はじめに

「APIキーをチャットに貼り付けて大丈夫なの？」「自分のThreadsアカウントの情報が外に漏れないか心配」——そんな不安を持つ方のために、このシステムがどのようにあなたの情報を守っているかをわかりやすく説明します。

---

## 1. APIキーはどこに保存されるの？

セットアップ中にGeminiやGitHubのAPIキーをチャットに貼り付けますが、それらは即座に **あなたのPC内にある `.env` ファイル** に書き込まれます。

```
あなたのPC
└── .env ← ここにだけ保存される（外には出ない）
```

**インターネット上のサーバーには一切送られません。** Claude Codeはあなたのパソコン上でローカルに動いているツールなので、入力した情報がクラウドに保存されることはありません。

---

## 2. APIキーがGitHubに上がることはありますか？

**ありません。**

このシステムには `.gitignore` という仕組みが設定されており、`.env` ファイルは自動的にGitHubへのアップロード対象から除外されます。仮に「GitHubに送る」操作をしても、APIキーが含まれる `.env` だけはスキップされる設計になっています。

---

## 3. AIが投稿する際、Threadsのアカウント情報は安全？

Threadsのアクセストークン（投稿権限のカギ）も `.env` ファイルにのみ保存されます。

AIが投稿を実行する際の流れはこうです：

```
あなたのPC内の .env からトークンを読み取る
        ↓
Threads API に直接リクエストを送る
        ↓
投稿完了
```

トークンがあなたのPC外に出るのは「Threads APIへのリクエスト時のみ」であり、その通信はThreads公式のHTTPS（暗号化通信）で保護されています。第三者が途中で盗み見ることはできません。

---

## 4. GitHub Actionsで自動実行される場合は？

スネイプの監視など一部の機能はGitHub Actionsで自動実行されます。この場合、APIキーは **GitHub Secrets** という専用の秘密情報管理機能に保存します。

GitHub Secretsの特徴：
- ログや画面上に値が表示されることは一切ない
- リポジトリの管理者であっても、一度登録した値を見返すことはできない
- GitHubのサーバー上で暗号化されて保管される

---

## 5. 他のメンバーに自分の情報が見えることはある？

**ありません。**

このシステムは各メンバーが**自分のGitHubリポジトリ**を使って運用します。他のメンバーのリポジトリにアクセスする手段はなく、投稿内容・APIキー・アカウント情報が他メンバーに見えることはありません。

---

## 6. 月次認証キー（.keyファイル）の仕組み

毎月配布される認証キーファイルは：

- `.gitignore` により、GitHubには**絶対にアップロードされない**設定になっています
- 有効期限が切れると自動的にシステムが停止します
- キーファイルを他人と共有しても、GitHubリポジトリへのアクセス権がなければシステムは動きません

---

## 7. このシステムが収集・利用するデータについて

このシステムが扱うデータの利用目的は以下のみです：

| データ | 用途 |
|---|---|
| Threads投稿内容 | あなたのアカウントへの投稿のみ |
| YouTubeの公開動画情報 | ネタ収集（閲覧のみ・書き込みなし） |
| GitHub Issues | エージェント間の作業記録（あなたのリポジトリ内のみ） |
| Google Sheetsのデータ | エンゲージメント記録（設定した場合のみ） |

**収集したデータを第三者に送信・販売することは一切ありません。** すべてあなた自身のアカウントとリポジトリの中で完結します。

---

## 8. もしAPIキーが漏れてしまったら？

万が一APIキーが流出した場合でも、以下の手順で即座に無効化できます：

| キーの種類 | 無効化方法 |
|---|---|
| Gemini API Key | Google AI Studio で削除・再発行 |
| GitHub Token | GitHub設定画面でトークンを削除 |
| YouTube API Key | Google Cloud Console で削除 |
| Threads Token | Meta Developer Portalで無効化 |

キーを無効化した後、新しいキーを `.env` に設定し直すだけで復旧できます。

---

## まとめ

このシステムは以下の原則で設計されています：

1. **すべての機密情報はあなたのPC内にのみ保存** — インターネット上のサーバーには送られません
2. **GitHubには秘密情報は上がらない** — `.gitignore` と GitHub Secrets で自動保護
3. **他のメンバーとデータは完全に分離** — 各自が独立したリポジトリで運用
4. **データはあなた自身のために使うだけ** — 第三者への提供は一切なし

わからないことがあればコミュニティのDiscordでいつでも質問してください。
