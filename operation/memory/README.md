# 外部メモリ：GitHub Issues 運用ガイド

このフォルダは、ローカルの外部メモリ管理ドキュメントです。
実際のデータは **GitHub Issues** に保存されます。

---

## 📋 Issueラベル体系

| ラベル名 | 用途 | 作成者 |
|---|---|---|
| `daily-operation` | 毎日の運用ループ進行管理 | ハリー（自動） |
| `投稿実績ログ` | 投稿ごとのエンゲージメントデータ | ロン（自動） |
| `承認待ち` | 人間承認待ちの投稿案 | マルフォイ（自動） |

---

## 📂 Issue構成

### 毎日作成されるIssue（`daily-operation`ラベル）

```
タイトル: 【運用ループ】YYYY-MM-DD - [テーマ名]

コメント順（エージェントが順番に書き込む）:
1. ハーマイオニー: リサーチ＆ブリーフィング
2. ルーナ: 投稿案3案
3. マルフォイ: 審査結果＆承認申請
4. [オーナー]: 「承認」コメント
5. ロン: 投稿完了報告
6. ロン: 計測結果（24時間後）
```

### 投稿実績ログIssue（`投稿実績ログ`ラベル）

毎日のIssueが24時間後にクローズされると、自動的に `投稿実績ログ` ラベルが付与されます。
ハーマイオニーが過去データを分析する際に参照します。

---

## 🔍 エージェントがIssueを使う方法

```python
from scripts.utils.github_issues import GitHubIssues

gh = GitHubIssues(token, repo)
issue = gh.get_or_create_today_issue()  # 今日のIssue取得
gh.add_comment(issue.number, "コメント内容")  # コメント追加
```

---

## ⚠️ 承認フロー

1. マルフォイが `承認待ち` ラベルを付与
2. Makeがオーナーのスマホに通知
3. **オーナーがIssueのコメント欄に「承認」と入力**
4. ロンが承認コメントを確認してから投稿実行

**承認コメント以外では投稿されません。**
