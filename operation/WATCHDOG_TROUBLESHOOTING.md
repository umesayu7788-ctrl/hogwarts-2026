# GAS Watchdog トラブルシューティング

> このファイルはGAS Watchdog警報を受け取ったときに参照するフローチャートです。

---

## ⏰ まず確認：今は何時ですか？

| 時刻 | あなたがすること |
|---|---|
| 6:30〜7:00 | **待つだけ**。自動復旧が走っています（承認通知を待つ） |
| 7:00〜8:00 | 承認通知がまだ来ない場合のみ、以下のフローチャートへ |
| 8:15 以降 | 今日の投稿は手動で対応が必要な可能性あり |

---

## 🔍 フローチャート（7:00以降も通知が来ない場合）

```
承認通知が来ない
      ↓
GitHub Actions ログを確認
https://github.com/[あなたのリポジトリ]/actions
      ↓
daily-cycle が実行されているか？
      ↓
【YES】実行中・完了
→ ログ内でどのステップでエラーになっているか確認
→ 下の「エラーパターン別対処」を参照

【NO】実行されていない
→ GAS側のログを確認（下記）
→ または手動でdaily-cycleを実行（Actionsページから「Run workflow」）
```

---

## 🔧 エラーパターン別対処

| パターン | 症状 | 対処 |
|---|---|---|
| GAS未発火 | Actionsに何も記録なし | GAS側のトリガーが削除されていないか確認。SETUP_GAS_TRIGGER.md参照 |
| Gemini 429 | Actions ログに「429 Resource Exhausted」 | 60分待ってから手動実行 |
| GitHub Token失効 | Actions ログに「401 Bad credentials」 | GitHubトークン再発行 → Secrets更新 |
| Threads Token失効 | 投稿ステップでのみエラー | Threadsトークン手動更新（access_HOG-MM.keyを参照） |
| 不明なエラー | 下記テンプレートでClaudeに報告 | Claude Codeを開いてエラーログを貼り付け |

---

## 📋 Claude Code への報告テンプレート

以下をコピーしてClaude Codeに貼り付けてください：

```
今日のGAS Watchdog警報に対応しています。
以下のActionsログを確認してください：

エラー内容: [ここにActionsログのエラー行を貼り付け]
発生時刻: [HH:MM]
```
