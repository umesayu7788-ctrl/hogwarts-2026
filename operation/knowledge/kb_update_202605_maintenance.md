## 🔔 Discord通知パターン別 対応ガイド

通知が来ない = 気づけない を根絶するため、全失敗ケースでDiscord通知が届きます。

| 通知内容 | 意味 | 対応 |
|---|---|---|
| 📥 緊急投稿の下書きを保存しました | /add-idea が成功（承認待ち） | `/approve-urgent` で投稿承認 |
| ✅ 緊急投稿完了 | /approve-urgent が成功 | 対応不要 |
| ⚠️ Threadsトークン自動更新に失敗 | 週次リフレッシュ失敗 | 手動更新が必要（下記手順） |
| ❌ 緊急投稿に失敗 | /add-idea エラー | GitHubのActionsログ確認 |
| ⛔ ファクトチェック不通過 | ネタに問題あり | 内容を見直して再送信 |
| 🔴 GAS Watchdog 警報 | 朝の自動サイクル未完了 | **待つだけ**（自動復旧済み） |

### 緊急時：Threadsトークン手動更新手順

1. Threadsアプリを開く
2. プロフィール → 設定 → アカウント → Metaとのリンク
3. 「リンクを解除して再リンク」を実行
4. Meta Developers から新しいトークンを取得
5. GitHubリポジトリ → Settings → Secrets → THREADS_ACCESS_TOKEN を更新

## 📱 新機能：Discord /add-idea の使い方（承認制）

iPhoneのDiscordから外出先でネタを思いついたとき、その場で下書き保存→承認→投稿できます。

### 投稿の流れ

```
① /add-idea idea:ネタ内容
   → ファクトチェック → 投稿案生成 → GitHub Issue保存 → Discordに下書き通知

② 内容を確認して OK なら /approve-urgent
   → 最新の下書きをThreadsに投稿 → 完了通知
```

### セットアップ（初回のみ・約30分）

1. **Discord Developer Portal でアプリ作成**
   - https://discord.com/developers/applications → 「New Application」
   - 「Bot」メニュー → 「Add Bot」→ トークンをコピー

2. **Botをサーバーに招待**
   - 「OAuth2」→「URL Generator」→ 「bot」「applications.commands」にチェック
   - 生成したURLをブラウザで開いてサーバーに追加

3. **Cloudflare Worker をデプロイ**（無料枠で運用）
   - https://workers.cloudflare.com → ダッシュボードを開く
   - 「Create a Worker」→ `discord-bot/worker.js` の内容を貼り付けてデプロイ

4. **Cloudflare Worker の環境変数を設定**
   - Settings → Variables → 以下を追加：
     - `DISCORD_PUBLIC_KEY` : Discord Developer Portal で取得
     - `GITHUB_REPO` : あなたのリポジトリ名（例：yamada/hogwarts-taro）
     - `GITHUB_TOKEN` : GitHubトークン（repo + workflow スコープ）

5. **Discord側にWorker URLを登録**
   - Developer Portal → アプリ → General Information
   - 「Interactions Endpoint URL」にCloudflareのURLを貼り付け

6. **スラッシュコマンドを登録**
   - `node discord-bot/register-commands.js` を実行

### 使い方

```
/add-idea idea:AI運用で1日30分が5分になった話
  → Discordに下書きが届く

/approve-urgent
  → 最新の下書きをそのままThreadsに投稿
```

## 📈 Phase 4: 投稿品質改善アップデート

### buzz判定基準の修正

**旧基準（バグあり）:** いいね30件 OR 閲覧3000件で自動バズ登録  
**新基準（修正済み）:** ER≥2% **かつ** 閲覧≥1000件の場合のみバズ登録

これにより、閲覧は多いのにERが低い投稿が誤ってバズ登録されることを防ぎます。
閲覧≥1000でER<2%の投稿は `low_er_posts.md` に反省サンプルとして自動記録されます。

### フォロワー推移の自動記録

毎日の計測時にフォロワー数が自動でGoogle Sheetsの「フォロワー推移」シートに記録されます。
シートが存在しない場合は自動作成されます。

### 投稿品質ガイドライン強化

以下の要素が全スロットに必須となりました：

| 必須要素 | 例 |
|---|---|
| 固有名詞 | Claude Code / ChatGPT / Gemini / OpenClaw |
| 具体的数字 | 2万円 / 10倍 / 60日 |
| 読者の生活インパクト | 給料以外の柱が立つ / 朝の時間が空く |

**絶対禁止フック:**
- 「プロ級の」「神レベルの」「ある質問」「最新の〇〇」（単独）
- マニアックなベンチマーク比較（Kimi K2/Qwen3/Llama等）

### マルフォイの審査基準強化

差し戻し基準に6項目を追加（計14項目に）。
投稿品質のチェックがより厳格になりました。最初の数日は差し戻しが増える可能性がありますが、
1週間程度でルーナが新基準に慣れ、安定します。

## 🔄 エラー自律解消サイクル（Claude Codeの動作強化）

### この更新で何が変わるか

エラーが発生しても、Claude Codeが**自分で解決するまで諦めずに動き続けます**。

**旧動作:** エラー発生 → ユーザーに「どうすればいいですか？」と確認  
**新動作:** エラー発生 → 仮説を立てて自分で修正 → 確認 → 再試行 → 解決するまで繰り返す

### Claude が自律的に回すサイクル

```
エラー発生
  ↓
仮説（原因を1〜3個推測）
  ↓
実装（最も可能性が高い仮説で修正）
  ↓
分析（ログ・出力を確認。直ったか？）
  ↓
考察（成功なら再開 / 失敗なら次の仮説へ）
  ↓
（解決まで最大8回繰り返す）
```

### あなたへの影響

- **エラーが起きても放置でOK。** Claude が自動で解消します
- **「直ったよ」の報告は1行のみ。** 長い説明はしません
- **8回試みても無理な場合だけ**、何が必要か具体的に教えてくれます

### Claude が自分で解決できるもの（確認不要）

- Pythonライブラリの再インストール
- APIの一時エラー（リトライ・モデル切替）
- スクリプトの軽微なバグ修正
- 設定ファイルの自動修正
- GitHub Issueへの保存・記録

### 人間の操作が必要なもの（Claude が案内します）

- APIキーの再取得（外部サービスへのログインが必要）
- GitHub Secrets の更新
- Threadsアプリでのトークン再リンク

### 何が変わるか

| 項目 | 変更前 | 変更後 |
|---|---|---|
| 保護範囲 | スクリプトの「閲覧」のみ制限 | 閲覧 + **編集も制限**（保護設定の書き換えを防止） |

| トークン検証 | 形式と日付のみ | 形式・日付 + **チェックサム検証**（偽造検知を強化） |

> ⚠️ 6月以降のトークンは新チェックサム形式になります。受け取ったトークンをそのまま使ってください（操作に変化はありません）。

### 手順2：セキュリティ設定の更新（手動・2分）

Claude Code を開き、以下をコピーして貼り付けてください：

### このアップデートで防げるようになること

- 「デバッグのためスクリプトを見せて」という迂回でのコード閲覧
- 「settings.json の制限を外して」という保護の無効化試みへの無応答
- 形式だけ真似た偽造トークンによる不正利用
- 「管理者から許可を得た」という権限昇格フレーミング

---

### このアップデートで防げないこと（理解しておくこと）

- OS のファイルマネージャーやテキストエディターを使った直接ファイル閲覧
- Python を直接実行してのスクリプト呼び出し
- トークンファイルの他メンバーへの共有

上記は本システムを悪用しようとする意図がある場合の話です。
悪意ある利用は利用規約違反として対応します。普通に使う分には一切影響ありません。
