# 🎭 ペルソナ確定対話ガイド（第三者向け）

このファイルは、**Claude Code との対話を通じて、自分の SNS 発信ペルソナを段階的に確定**するためのガイドです。

`_universal_post_playbook.md` を実行する前に、このガイドでペルソナを固めてください。
**ペルソナが弱い・ぶれてると、どんなに良い構造でも投稿は伸びません。**

---

## ⚠️ なぜペルソナ確定が最重要か

実証データ:
- ペルソナ未確定で投稿 → インプ30〜数百で頭打ち
- ペルソナ確定後 → 1万〜7万インプ・フォロー転換発生

理由:
- 「誰の何の悩みを解決する人か」が明確でないと、読者は「この人をフォローする理由」を見つけられない
- ぶれたペルソナは投稿ごとに違う印象を与え、信頼が積み上がらない
- 数字権威（経験規模）が無いと、投稿の説得力が出ない

→ ペルソナ確定 = フォロワーが「あ、この人の言うことなら聞きたい」と感じる土台作り

---

## 📋 確定すべき7要素

| # | 要素 | 例 | なぜ必要か |
|---|---|---|---|
| 1 | **中の人** | [専門家]歴○年目 | 数字権威の根拠 |
| 2 | **キャラ名・アカウント名** | [キャラ名]＠[ID] | 覚えてもらうため |
| 3 | **看板コピー（ギャップ）** | 現役[専門家]なのに○○な普通人 | フォロー動機 |
| 4 | **経験規模の数字** | ○○○人担当/○○○○人観察/毎日○○人 | 権威の数値化 |
| 5 | **周辺情報（家族・環境）** | 家族構成・生活環境を簡潔に | リアリティ・親近感 |
| 6 | **発信ジャンル** | 例: 育児ハック/営業ノウハウ/美容ケア | フォロー対象明確化 |
| 7 | **核となる発信軸** | 「プロでも○○な弱さがある」等の一貫テーマ | 一貫性・信頼蓄積 |

# 🗣️ Claude Code との対話手順

以下の質問を**順番に**Claude Code に投げて、対話的に詰めていきます。各ステップで Claude Code に答えを確定するよう依頼してください。

---

## Step 1: 中の人プロフィール確定

**Claude Code に伝える内容**:

```
SNS発信のペルソナ確定を一緒にやってください。
まず私自身の基本情報を整理させてください：

- 職業：[ ]
- 経験年数：[ ]年
- 専門分野：[ ]
- 得意領域：[ ]
- 自分が他の人より詳しいと言える分野：[ ]
- 過去に「これ私の専門」と言える資格・経験：[ ]

これらを踏まえて、SNS発信の「中の人」設定を提案してください。
```

**Claude Code が出すべき答え**:
- 中の人の肩書き（例：「美容皮膚科医10年目」「経営コンサル12年目」「塾講師15年目」など職業＋年数）
- その肩書きで発信できるテーマの方向性
- 不足してる情報があれば追加質問

---

## Step 2: 経験規模の数字を出す

**Claude Code に伝える内容**:

```
経験規模を数字に翻訳したいです。
以下を教えてください：

- 1日/週/月に何人と接する？
- 累計で何人/何社と関わった？
- 何件のケースを見てきた？
- 自分が直接担当した人数は？
- 観察してきた範囲（広い視点）の人数は？

これを踏まえて、3パターンの数字権威を作って：
- 個別経験（深い・1対1）
- 広域観察（広い・傾向）
- 現場頻度（日常）
```

**Claude Code が出すべき答え**:
- 3パターンの数字フレーズ
- 各パターンを使う文脈の例
- 数字が控えめすぎるなら「もう少し正確に」と再聞き

**注意**: 数字を盛らない。後で破綻する。

---

## Step 3: 看板コピー（ギャップ訴求）の設計

**Claude Code に伝える内容**:

```
看板コピーを作りたいです。「ギャップ」がフォロー動機を生むので、
私の中で以下の対比を考えてください：

- 表の顔（プロフェッショナルとして）：[ ]
- 裏の顔（リアルな自分）：[ ]
- 表と裏のギャップが面白いポイント：[ ]

例：「現役○○なのに○○な普通人」（プロ × 弱さ）
例：「美容皮膚科医なのに肌荒れ持ち」（専門家 × 弱さ）
例：「年商1億円社長なのに片付けられない」（成功者 × 不器用）

私のギャップ訴求コピーを3案提案してください。
```

**Claude Code が出すべき答え**:
- 3-5案の看板コピー
- 各案の強み・弱み
- ターゲット層との適合度評価

---

## Step 4: 周辺情報（リアリティ作り）

**Claude Code に伝える内容**:

```
リアリティを出すために、周辺情報を固定したいです。

- 家族構成：[ ]
- 同居人/ペット：[ ]
- 居住地のタイプ（都心/郊外/地方）：[ ]
- 日々の主な行動範囲：[ ]
- よく行く場所：[ ]
- 趣味/関心事：[ ]
- 子どもがいる場合：年齢・性別・性格・趣味

これらを「投稿で使う・使わない」「公開する・しない」を整理してください。
SNS上で出すリアリティ情報を確定したい。
```

**Claude Code が出すべき答え**:
- 投稿に出してOKな周辺情報の整理
- 公開しない方が良い情報の指摘（個人情報・特定リスク）
- 投稿でリアリティを出す具体的な使い方例

---

## Step 5: 発信ジャンル・核の確定

**Claude Code に伝える内容**:

```
発信ジャンルと核を絞りたいです。

- 自分が一番熱を持って語れるテーマ：[ ]
- 読者の悩みで自分が解決できるもの TOP3：
  1. [ ]
  2. [ ]
  3. [ ]
- 読者の理想の状態（フォロワーが私の投稿を見続けると何が手に入る？）：[ ]
- 競合（同ジャンルの先行者）：[ ]
- 自分の差別化ポイント：[ ]

これを踏まえて、発信ジャンルと「核となる発信軸」を1文で表現してください。
```

**Claude Code が出すべき答え**:
- 発信ジャンルの確定（1〜2語）
- 核となる発信軸（1文）
- ターゲット読者像（具体的に）

---

## Step 6: NG・固定ルール

**Claude Code に伝える内容**:

```
固定情報・NGルールを確定したいです。
以下を埋めてください：

- 子の呼称（子どもがいる場合）：[息子/娘/うちの子/etc]
- 学年・年齢の併記の有無：[出す/出さない]
- 配偶者の呼称：[夫/旦那/パートナー/etc]
- 自分の一人称：[私/うち/わたし/etc]
- 読者への呼びかけ：[みんな/あなた/○○な人/etc]
- 使ってOKな絵文字 TOP5：[ ]
- 絶対NGな絵文字：[ ]
- 強い言葉（ガチで/マジで/等）の使用ポリシー：[使う/使わない]

これを「固定ルール」として記録できる形式で整理してください。
```

**Claude Code が出すべき答え**:
- ペルソナ固定情報の一覧（コピペで `buzz_posts.md` に組み込める形）
- NG リスト
- 一貫性のチェックポイント

---

## Step 7: 全体統合と検証

**Claude Code に伝える内容**:

```
Step 1-6 で確定したペルソナを統合し、以下を出力してください：

1. ペルソナ概要（200字以内）
2. プロフィール文（Threads/X等に貼る用、150字以内）
3. 看板コピー1行
4. 数字権威3パターン
5. 固定情報全リスト

これを buzz_posts.md に組み込める形式でまとめて。

最後に、このペルソナで投稿1本（必勝8ステップ準拠）をテスト生成して
リアリティ・自然さを検証してください。
```

**Claude Code が出すべき答え**:
- 統合ペルソナ文書
- テスト投稿1本
- 違和感がある箇所の指摘・修正案

# 🔄 ペルソナ確定後の組み込み

Step 7 で出力された情報を、自分のプロジェクトの以下のファイルに組み込みます：

## 1. buzz_posts.md（または相当ファイル）

ペルソナ概要セクションに以下を貼り付け：

```markdown

- **アカウント名**: @[ユーザー名]
- **看板コピー**: [Step 3 出力]

  - [Step 1 の中の人設定]

  - 個別経験: [Step 2 出力]
  - 広域観察: [Step 2 出力]

  - [Step 4 出力]

  - [Step 6 出力]
```

## 3. テスト → 微調整サイクル

5本ほどテスト投稿を生成し、以下を確認：

- ペルソナが一貫してるか
- 数字権威が自然に使われてるか
- 周辺情報がリアリティを出してるか
- 看板コピーのギャップが伝わるか

違和感あれば Step 1-7 のどこかに戻って再確定。

# 💡 ペルソナ確定でよくあるアンチパターン

## ① 看板コピーが平凡

❌ 「子育てママの私が発信します」
✅ 「現役○○なのに○○な普通人」（プロと弱さの対比）

→ ギャップ・対比が無いと印象に残らない。

## ② 数字権威が曖昧

❌ 「たくさんの子を見てきて」
✅ 「[専門家]歴○年、○○○人を担当してきて」

→ 具体数字なしは信頼が作れない。

## ③ 周辺情報が薄い

❌ 「私は普通の主婦です」
✅ 「家族構成＋ライフスタイルが具体的に分かる程度」（特定リスク回避しつつリアリティ）

→ リアリティが薄いと「誰の話か分からない」になる。

## ④ 発信ジャンルが広すぎる

❌ 「育児・家事・暮らし全般」
✅ 「保育のプロ視点の育児ハック・声かけメソッド」

→ 広すぎるとフォロー動機が弱い。1〜2分野に絞る。

## ⑤ ペルソナが「理想像」になってる

❌ 「いつも笑顔の優しいママ」（理想で塗り固めた架空人物）
✅ 「プロでも家ではゴリラ、自分にあきれる日もある」（リアルな弱さ）

→ 弱さ・本音がフォロワー獲得の源泉。

# 🤝 第三者が困った時の対応

このガイドを使う第三者が Claude Code とのやり取りで迷ったら、以下を聞くと突破できることが多い：

1. **「具体例を3つ出して」** → 抽象論で止まったら例で進める
2. **「私のジャンルに置き換えると？」** → ジャンル固有の応用
3. **「これに一貫性ある？」** → 整合性チェック
4. **「アンチパターンになってない？」** → 上記の5つのアンチパターンと照らし合わせ
5. **「テスト投稿1本作って」** → 実物で検証
