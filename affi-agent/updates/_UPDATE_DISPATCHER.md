# 🚀 アップデート適用プロンプト（顧客が Claude Code に貼るだけ）

このファイルの**「📋 ここから下を Claude Code にコピペ」**セクションをそのまま Claude Code に貼ってください。
Claude Code が自動で必要なアップデートを判定し、対話的に適用してくれます。

---

## 📋 ここから下を Claude Code にコピペ ↓↓↓

```
私のプロジェクトに updates/ フォルダ経由でアップデート配布が届いています。
以下の手順で適用してください：

【Step 1】updates/_MANIFEST.md を読む
- 配布されているアップデートの一覧と各更新の概要を把握
- 各アップデートの「目的」「対象」「前提条件」「変更ファイル」を確認

【Step 2】私の現状をヒアリング
以下を質問して、私の状況を確認してください：
- 現在の運用フェーズ（新規/既存運用中/数ヶ月稼働中）
- ペルソナ確定状況（完全確定/部分確定/未確定）
- 自動化レベル（手動/半自動/完全自動）
- 直近の課題（インプ伸び悩み/フォロー伸び悩み/収益化未着手/etc）

【Step 3】適用すべきアップデートを提案
- _MANIFEST.md と私の現状を照合
- 「今適用すべき」「後で良い」「適用不要」に分類
- 各アップデートの優先度と理由を提示

【Step 4】私の承認を待つ
- 私が「これを適用」と指示したアップデートだけ実装
- 適用順序を提案（依存関係がある場合）

【Step 5】選ばれたアップデートを順次実装
- updates/packages/ 配下の該当ファイルを読み、手順通り実装
- 各アップデート完了ごとに「完了」報告
- 既存ファイルへの統合は「追記・統合」を原則とし、上書きしない

【Step 6】全体完了報告
- 適用したアップデート一覧
- 変更されたファイル一覧
- テスト結果（テストが含まれるアップデートの場合）
- 未適用のアップデート（理由付き）

【重要な原則】
- 「全部適用」しない。私の現状に必要なものだけ
- 既存のペルソナ・運用ロジックは尊重し、勝手に変更しない
- 各ステップで承認を求める
- 適用済みアップデートは updates/_APPLIED.md に記録（後で重複適用を防止）

ではまず Step 1 から始めて、利用可能なアップデート一覧を私に見せてください。
```

# 🛠️ 配布側の運用ルール（user 用メモ）

## 新しいアップデートを追加するとき

1. `updates/packages/NNN_xxx.md` を新規作成（NNNは3桁の連番）
2. `updates/_MANIFEST.md` に新エントリ追加
3. ファイル名・場所・依存関係をマニフェストに正確に記載
4. テストで動作確認してから配布

## マニフェスト更新時のポイント

- 各アップデートは**独立**してること（他のアップデートに依存しない）
- 依存がある場合は明示（例: 002 は 001 の後で適用）
- 「対象」を明確に（誰が適用すべきか）
- 既存ファイルへの影響範囲を明記

## 顧客への配布フロー

1. 新アップデートのファイルを追加
2. `_MANIFEST.md` 更新
3. ZIPまたはGitHubで配布
4. 顧客は `_UPDATE_DISPATCHER.md` の中身を Claude Code に貼るだけ
5. Claude Code が manifest を読んで自動で「これが新しいよ」と提案
