## 🎯 結論：アフィリ投稿は「親 + ツリー1個」の2投稿構造（ジャンル横断）

ツリーを長くするほど離脱が増える。バズ＋成約しているアカウントは多くの場合 **2投稿構造** を採用している。

```
┌──────────────────────────┐
│ 親投稿（1/2）             │
│ ─────────────────────── │
│ ・強烈フック1〜2行        │
│ ・体験/権威/根拠（短文）  │
│ ・商品名は登場OK（自然に）│
│ ・写真や動画があれば添付  │
│ ※PR表記・リンクは絶対NG  │
└──────────┬───────────────┘
           │ reply
           ▼
┌──────────────────────────┐
│ ツリー1（2/2・最終）      │
│ ─────────────────────── │
│ ・続き or 補足ひとこと    │
│ ・「pr」または「PR」単独行│
│ ・楽天/Amazonアフィリリンク│
└──────────────────────────┘
```

→ `===THREAD===` 区切りは**1個だけ**。ライターは2ブロックのみ生成する。

## 🔥 親投稿（1/2）のフック設計（ジャンル横断テンプレ）

### 損失回避型（最強・拡散しやすい）
```
マジで広がって欲しくないけど、{{ターゲットの目標}}したいなら〇〇試して
```

### 権威即出し型
```
{{あなたの肩書き・経歴}}の私が断言する。{{結論}}
```

### ターゲット呼びかけ型
```
{{ターゲットの悩み}}で限界の{{あなた}}へ
```

### 体験告白型
```
実は私、{{自分の悩み・状況}}だったから話題の{{商品名}}試してみた。
```

### 結論直撃型
```
{{結論を1行で}}。
```

→ 自分のジャンルに合うフック型を **3〜5つ** 選んで重点運用すると効率良い。

## 📐 親投稿の本文ルール

- **3〜5行が理想**（参考アカウント全例で5行以内）
- 体験ベースの具体性
- 権威の見せ方：1行目に肩書きをサラッと混ぜる（buzz_posts.md声定義から）
- 商品の効果・根拠を1〜2行で（専門用語はごく簡単な翻訳で）
- **CTAなし**（ツリーで完結する設計なので、親では引きを残すだけ）

### 親投稿で書いてはいけないもの
- 【PR】 / `#PR` / `pr` / `#広告` ← どれもNG（最終ツリーに置く）
- アフィリリンク ← NG
- 「リンクから詳細見てね」「プロフィールから」のCTA ← NG

---

## 📐 ツリー1（最終ブロック）のルール

- **2〜4行**で簡潔に
- 親投稿の続き or 補足（弱点・注意・別の使い方など両面提示）
- 「**pr**」または「**PR**」を**単独行**で配置（小文字 `pr` も可）
- 改行してアフィリリンク（`a.r10.to/xxx` または `amzn.to/xxx` の短縮形でOK）

### 参考フォーマット
```
{{補足や注意点を1行で・両面提示}}
ちなみに〇〇な点だけ気をつけてね！pr
{product_url}
```

→ pr表記の位置は「文末の `pr`」でも「単独行 `PR`」でもどちらもOK。**自然に置けることが最優先**。

## ✅ 学習する点 / ❌ パクらない点

### ✅ 学習する（構造・型・誘導）
- 2投稿構造（親+ツリー1）
- 損失回避型・権威即出し型のフック型
- ラフな口語（「マジで」「ガチで」など）
- pr表記の位置（文末 or 単独行）
- リンクは短縮形のみ
- 弱点・注意点の併記（両面提示）

### ❌ パクらない（オリジナリティ確保）
- 文章のコピペ・近すぎる言い回し
- 商品ジャンル丸かぶり（同じ商品を真似してすぐ紹介しない）
- 同じ肩書きの言い方（自分の固有性で表現する）
- 写真・動画の構図（自前の素材があれば自前を）

## 🚨 コンプライアンス遵守ライン

参考アカウントもステマ規制対応として `pr` を必ず置いている。
→ こちらも**ツリー1（最終ブロック）に必ず `pr` または `PR` を置く**。親投稿に置いてはいけない理由は、フックの強さを削ぐから。
→ 同一ツリー内に明示があれば消費者庁ガイドラインの「明瞭性」要件は満たす。
