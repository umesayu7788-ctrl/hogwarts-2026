> **このナレッジは全エージェント（特にライター・コンプライアンス・校閲・ファネル設計・ロン投稿）が参照すること。**
> 違反すると **アカウント停止**＋**過去投稿も遡及審査**される。報酬没収リスクあり。

| # | 禁止行為 | 公式原文 |

## ✅ 守るべきルール

### 認定SNSのみ利用可
- Threads は **2023/07/13 から認定SNS**（公式に許可）
- 各SNSのスパム定義も遵守すること

### コンテンツ品質
- **同じ内容を繰り返さない**：商品が同じでも、訴求角度・体験談・季節文脈を毎回変える
- **内容の薄い投稿NG**：単なる「これいいよ！」だけは禁止。必ず体験・根拠・気づきを添える
- **過去投稿との重複も審査対象**：3ヶ月遡って類似投稿があるかチェック

### ハッシュタグ
- 乱用禁止。**1投稿あたり3〜5個まで**
- **必ず投稿内容に関連したもの**のみ
- アフィリ投稿には `PR`（単独行）/ `#PR` / `#広告` のいずれかを**ツリー最終ブロック内**に必ず配置
  - 親投稿には絶対にPR表記を入れない（フック・共感に集中するため）

### ステマ規制（景表法・併せて遵守）
- アフィリ投稿には **PR / #PR / #広告** を**ツリー最終ブロック（リンク添付ブロック）に明示**
- 「PR」単独行 → 楽天アフィリリンク の順で配置するのが推奨フォーマット
- これは楽天規約とは別に、消費者庁ガイドラインで義務化（2023/10/施行）
- ※ 親投稿にPR表記がなくても、同一ツリー内（投稿者が直接連結したreply）に明示されていればOK

## 🎯 このパイプラインでの遵守メカニズム

### 1. 「同じ内容の繰り返し」防止
- **ファネル設計担当**: 週次計画で**毎日違う商品・違う訴求角度**を割り当て
- **ライター**: 既存投稿（過去14日分）と内容類似度チェック → 類似度70%超は差し戻し
- **校閲**: 過去投稿との重複検知（タイトル・冒頭フック・商品名のいずれかで）

### 2. 「内容の薄い投稿」防止
- **ライター**: 全投稿で **体験エピソード／根拠／具体的な変化** のいずれかを含めることを必須化
- **校閲**: 「本文100字未満」「フック1文だけ＋商品リンク」のような薄い投稿は差し戻し

### 3. 「無関係キーワード/ハッシュタグ乱用」防止
- **ライター**: ハッシュタグ自体使用禁止（Threadsアルゴリズム上も逆効果）
- **コンプライアンス担当**: トレンドワード混入を検知して差し戻し

### 4. 「対価・見返り」防止
- **ライター**: 「コメント欄にリンク」「DMで紹介」等の表現を禁止語に登録
- **コンプライアンス担当**: 該当語を含む投稿を機械的にブロック

### 5. PR表記の自動付与
- **post.py / scheduled_post.py**: 楽天/Amazonリンク検知で **【PR】を冒頭に自動挿入**
- アフィリ投稿で PR表記なしは投稿前に強制付与

## 🔧 違反検知の運用ルール

### ライターは投稿生成時に必ず：
1. 過去14日の投稿履歴を `operation/weekly/funnel_*.md` から読む
2. 同じ商品・同じ訴求角度が連続していないか確認
3. 連続している場合、別の角度（季節・悩み軸・体験談切り口）に切り替え

### 校閲・コンプライアンス担当は審査時に必ず：
1. **本日の3スロット間で言い回し・冒頭フック・締め文の重複**がないか確認
2. 同SNS内（Threadsアカウント）で**3ヶ月内に類似内容**がないかGoogle Sheetsで照合
3. ハッシュタグの数・関連性チェック
4. 「コメント欄にリンク」「DM送ります」等の禁止表現スキャン

### 違反疑いの時：
- 即座に投稿を中止
- Discord通知で `@here` 警告
- ライターに角度変更を要求して再生成
